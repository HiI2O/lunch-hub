# ビルド & 修正

## 手順

1. ビルドを実行: `cd backend && npm run build`
2. エラー出力を解析し、ファイルごとにグループ化
3. 各エラーについて:
   - エラーコンテキストを読み取る（前後5行）
   - 原因を説明し、最小限の修正を適用
   - ビルドを再実行して解決を確認
4. 全エラー修正後、lint も実行: `npm run lint`

## ガードレール

以下の場合は **停止してユーザーに確認**:
- 修正が新しいエラーを導入した
- 同じエラーが3回の試行後も持続
- アーキテクチャの変更が必要と判断した場合

## サマリー

完了後、以下を表示:
- 修正されたエラー数
- 残りのエラー
- 変更したファイル一覧

NEVER 一度に複数のエラーをまとめて修正してはいけない。1つずつ確実に。
