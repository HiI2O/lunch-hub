# テストカバレッジ分析

## 手順

1. カバレッジ付きでテスト実行: `cd backend && npm run test:cov`
2. カバレッジレポートを分析
3. 80%を下回るファイルを特定
4. カバレッジ不足の各ファイルについて:
   - テストされていないコードパスを分析
   - 不足テストを生成（ハッピーパス、エラー、エッジケース、境界値）
5. 新しいテストがパスすることを確認
6. 前後のカバレッジ指標を表示

$ARGUMENTS が指定された場合、そのモジュールに絞って分析する。

## カバレッジ基準

- **80%以上**: 全コード
- **100%必須**: ドメインロジック、認証、金融計算
